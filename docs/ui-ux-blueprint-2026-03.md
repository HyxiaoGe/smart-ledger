# UI/UX 改造蓝图

更新时间：2026-03-08

## 目标

本轮 UI/UX 优化不追求“换一套皮肤”，重点是降低认知负担、缩短完成关键任务的路径，并统一交互节奏。

优先级顺序：

1. `/add`
2. `/`
3. `/records`
4. 统一设计语言与状态反馈

## 1. `/add` 页面

目标：把“填写很多字段”改成“先完成主输入，再按需补细节”。

设计原则：

- 第一屏只强调金额、分类、备注、保存
- 最近记录和高频金额要足够近，支持一键填充
- 日期、币种、支付方式、商家、子分类、产品全部放到次级区
- 保存后的状态反馈要非常明确

本轮已落地：

- 顶部增加录入引导区
- 主录入 / 快速填充 / 进阶信息三段式结构
- 高频金额前置到金额区
- 常用分类前置到主流程
- 最近记录改成更明确的右侧快速填充区
- 进阶信息折叠为次级区
- 保存按钮区域增加“当前准备保存”的即时摘要

后续可继续优化：

- 保存成功后增加“再记一笔 / 去看今天账单 / 撤销”
- 支持键盘优先的顺序录入
- 金额输入支持更强的数字键盘体验

## 2. 首页

目标：从“信息堆叠”改成“决策面板”。

建议方向：

- 顶部只保留核心指标、范围选择、币种切换、刷新状态
- 中部突出趋势和分类，不要所有模块同权
- Top 支出和热力图作为辅助判断，而不是并列主角
- AI 提示区弱化成辅助入口，不抢主数据层级

建议优先动作：

- 重新压缩顶部工具区和主指标的视觉层级
- 把图表区做成更清楚的主次结构
- 统一卡片内标题、说明、操作区间距

## 3. `records` 页面

目标：更适合“复盘”和“纠错”，而不是单纯看长列表。

建议方向：

- 顶部筛选条固定化，范围、分类、搜索要形成统一工作流
- 汇总、分类分析、明细列表形成更清晰的阅读节奏
- 列表里的编辑、删除、恢复尽量靠近行内完成
- 过滤后的列表结果要有更强的状态反馈

建议优先动作：

- 做统一 filter bar
- 减少模块之间的视觉断裂
- 强化空态、筛选中、同步中状态

## 4. 统一设计语言

后续全站 UI/UX 统一时，优先统一这些基础模式：

- 卡片：标题、说明、内容、底部操作区
- 状态：loading、success、error、empty、disabled
- 数据：金额、百分比、趋势箭头、日期标签
- 反馈：toast、确认框、局部刷新、全局刷新
- 工具栏：筛选、范围切换、快捷动作

## 5. 不建议优先做的事

- 全站强行换主题色
- 大量添加动画
- 重新做所有图表
- 先改设置页视觉而不改主工作流

## 6. 下一步建议

下一批 UI/UX 优化建议按这个顺序做：

1. 完成 `/add` 的交互回归与细节修正
2. 进入首页“决策面板化”
3. 进入 `records` 的筛选条和信息层级整理
4. 最后统一状态反馈组件
